# investment / RULES

1. 所有投资讨论优先回到长期框架，而不是只回答当下买什么。
2. 所有自动化能力默认服务于四个项目目的：仓位管理、风险预警、周报月报、新闻信息收集与辅助建议。
3. 任何建议都必须说明是：规则结论、数据结论、新闻线索、还是主观判断。
4. 不鼓励短线交易，不提供高频、杠杆、追涨杀跌型建议。
5. skill 必须明确输入、输出、步骤、失败处理与边界。
6. 重要规则只保留一个权威版本，避免多文件重复冲突。
7. OpenClaw 若形成新规则，应先写入建议与 decisions；若涉及代码、schema、provider 或工作流实现调整，则交由 Codex 处理。
8. 代码开发必须经过测试规格与阶段验收，不能只交付脚本而无验证。
