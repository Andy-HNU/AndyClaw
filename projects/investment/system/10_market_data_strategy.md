# 数据源策略

## 目标
为仓位管理、风险预警、周报月报、新闻收集提供稳定的数据输入。

## 主源策略
### 方案 A：东方财富 Choice 官方接口
适用：用户拥有权限时。
优点：接口体系完整，涵盖截面、序列、专题报表、宏观与资讯。
缺点：需要注册/试用或付费权限，部署前需确认账号与环境。

### 方案 B：社区封装的东方财富相关数据源
适用：没有官方权限时。
候选：
- efinance
- AKShare 中的东财相关接口

优点：接入成本低，便于快速验证。
风险：
- 页面/接口变动带来不稳定
- 限流、连接关闭、字段变更
- 不适合作为唯一数据源

## 备源策略
至少保留一个备用源。
建议：
- 若主源为官方 Choice，则备源为 AKShare 或 efinance
- 若主源为 efinance，则备源为 AKShare

## 抽象要求
无论底层使用哪个数据源，代码都必须只依赖统一抽象，例如：
- `get_realtime_quotes()`
- `get_history_quotes()`
- `get_fund_basic_info()`
- `get_index_snapshot()`
- `get_news_items()`

## 入库要求
- 原始抓取结果先标准化
- 再写入 SQLite
- 每次写入记录 `source` 与 `fetched_at`

## 失败退化
1. 主源失败 -> 切换备源
2. 备源失败 -> 使用最近一次有效快照
3. 快照过旧 -> 明确提示“数据可能已过期”

## 开发建议
先实现最小能力：
- ETF/基金最新净值或价格
- 指数快照
- 新闻抓取
再逐步扩展到宏观与行业数据
