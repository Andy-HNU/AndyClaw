# V2.1+ 盘中市场情绪分类扩展设计（Proposal）

## 目标
在现有四类（恐慌踩踏 / 分歧洗盘 / 追涨 / 区间震荡）基础上，扩展为更完整、可解释、可回测的盘中情绪框架。

## 设计原则
1. 对用户输出使用自然语言结论，不暴露内部标签。
2. 每个结论必须带证据（价格趋势、量能、振幅、回撤、尾盘形态）。
3. 采用概率/置信度表达，不输出绝对判断。
4. 支持降级：证据不足时只给“中性观察”。

## 情绪分类（建议 8 类）

### 1) 情绪高涨追涨（Momentum Chase）
- 典型特征：价格持续上行 + 成交量持续放大 + 回撤浅。
- 风险：冲高回落。
- 建议：不追高，等待承接确认。

### 2) 恐慌踩踏（Panic Selloff）
- 典型特征：价格快速下挫 + 放量 + 低位承接弱。
- 风险：流动性折价。
- 建议：先控仓位，等待止跌结构。

### 3) 分歧洗盘（Distribution/Washout）
- 典型特征：振幅大 + 量能抬升 + 收盘弱于盘中高点。
- 风险：方向未定、反复拉扯。
- 建议：降低追单，等尾盘确认。

### 4) 区间震荡（Range Chop）
- 典型特征：价格围绕中枢震荡，量能中性。
- 风险：假突破。
- 建议：观望，等突破方向。

### 5) 缩量阴跌（Low-volume Drift Down）
- 典型特征：价格缓慢下行 + 量能萎缩。
- 风险：趋势性走弱但不剧烈。
- 建议：降低进攻仓位，等待量价改善。

### 6) 缩量反弹（Weak Rebound）
- 典型特征：价格反弹但量能不足。
- 风险：反弹延续性差。
- 建议：当作修复看待，不急于追涨。

### 7) 放量突破（Volume Breakout）
- 典型特征：突破关键区间 + 明显放量 + 收盘保持强势。
- 风险：假突破回落。
- 建议：小步跟随，设置失效条件。

### 8) 高位滞涨（Exhaustion / Stall）
- 典型特征：价格创新高乏力 + 量能不再扩张或放量滞涨。
- 风险：获利盘回吐。
- 建议：降速加仓，防冲高回落。

## 证据字段（统一）
- `price_trend_pct`
- `volume_ratio`
- `amplitude_pct`
- `drawdown_from_high_pct`
- `close_position_in_range`（收盘在日内区间位置）
- `breakout_distance_pct`（对关键区间的突破距离）
- `matched_rules`
- `confidence_score`

## 输出层（用户可读）
- 结论：一句话自然语言
- 证据：2~4 个关键数据点
- 动作建议：继续观察/分批/减速/等待确认
- 失效条件：若 X 发生，则撤销当前判断

## 与现有架构关系
- 放在 `intraday_proxy_engine` + `signal-review` 层
- `daily-review` 只负责汇总与解释
- 报告不显示内部标签，只显示可读结论

## 实施优先级
P0: 完成 8 类规则和阈值配置化 + 回测夹具测试
P1: 增加尾盘确认规则（减少误判）
P2: 与月度双目标联动（战术仓位微调）
