# 完整系统工作流

## 总原则
工作流统一遵循：
1. 读取配置与数据库
2. 刷新或确认数据
3. 执行分析
4. 生成结论与报告
5. 写回 SQLite 与日志

说明：
- 若工作流中的代码能力已经存在，OpenClaw 负责调用、检查结果、形成验收记录。
- 若工作流中的代码能力尚未存在，OpenClaw 负责把缺口整理成 Codex 开发任务，而不是跳过实现前提继续假执行。

## 工作流 A：系统初始化
实现者：Codex
运行验收者：OpenClaw
1. 读取 `system/target_allocation.json`
2. 读取 `system/portfolio_state.json`
3. 初始化 SQLite
4. 创建基础表
5. 写入首份资产快照
6. 验证能查询到资产与目标配置

## 工作流 B：日常数据刷新
实现者：Codex
运行验收者：OpenClaw
1. 拉取行情数据
2. 拉取新闻数据
3. 写入 SQLite
4. 记录数据源、时间戳、状态
5. 若失败，切换备源
6. 若仍失败，保留旧快照并标记刷新失败

## 工作流 C：仓位检查
实现者：Codex
运行验收者：OpenClaw
1. 读取最新资产快照
2. 调用 `portfolio_analyzer`
3. 输出当前比例、高配/低配、偏离值
4. 写入分析快照

## 工作流 D：风险预警
实现者：Codex
运行验收者：OpenClaw
1. 读取最新仓位分析
2. 调用 `risk_monitor`
3. 检查集中度、偏离度、现金缓冲、数据异常
4. 输出风险等级与待观察事项
5. 写入风险信号表

## 工作流 E：每月定投建议
实现者：Codex
运行验收者：OpenClaw
1. 读取目标配置
2. 读取当前仓位分析
3. 调用 `monthly_planner`
4. 给出月度资金分配建议
5. 若接近再平衡阈值，附加“先用新增资金修正”说明
6. 写入建议表

## 工作流 F：再平衡专项检查
实现者：Codex
运行验收者：OpenClaw
1. 读取目标配置
2. 读取最新仓位分析
3. 调用 `rebalancing_engine`
4. 判断是否触发再平衡检查
5. 若未触发，继续观察
6. 若触发，先生成新增资金修正方案，再生成卖出讨论方案
7. 写入再平衡判断结果

## 工作流 G：周报
实现者：Codex
运行验收者：OpenClaw
1. 执行日常数据刷新
2. 生成本周仓位快照
3. 汇总本周新闻
4. 汇总本周风险信号
5. 调用 `report_generator`
6. 输出周报并入库

## 工作流 H：月报
实现者：Codex
运行验收者：OpenClaw
1. 执行月末数据刷新
2. 生成月度仓位分析
3. 执行再平衡专项检查
4. 生成下月定投建议
5. 汇总月度新闻与风险
6. 调用 `report_generator`
7. 输出月报并入库

## 工作流 I：GitHub 检索辅助
主导者：Codex
辅助者：OpenClaw
1. 明确要补的是哪一层能力：provider / storage / report / workflow
2. 依据 `agent/GITHUB_SKILL_DISCOVERY.md` 组织查询
3. 只抽取接口、结构、测试与失败处理思路
4. 记录参考来源与吸收点
5. 不直接引入大块未经审计代码
