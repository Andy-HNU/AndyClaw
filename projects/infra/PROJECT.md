# infra / PROJECT

## 项目目标
沉淀服务器、SSH、隧道、远程访问、自动化脚本、部署方案等实践经验，避免重复试错。

## 当前关注点
- SSH 登录与认证方式
- Windows 本地脚本封装
- 浏览器启动自动化
- 隧道与端口转发
- root 环境下的风险控制
- OpenClaw 与相关服务部署

## 当前状态
- 已完成：多轮服务器访问与脚本封装讨论
- 进行中：把这些讨论沉淀成可复用部署与运维规则
- 未完成：统一部署规范、回滚规范、环境检查脚本

## 风险点
- 直接以 root 执行高风险操作
- 脚本便利性高于安全性
- 不同环境下配置漂移

## 下一步
- 为常见部署与隧道场景形成标准脚本模板
- 给重要操作加风险说明与回滚方式
- 建立环境检查清单
