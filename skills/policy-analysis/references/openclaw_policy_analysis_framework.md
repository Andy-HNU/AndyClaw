# OpenClaw 政策与时事分析框架（七步版）

> 该文件为技能参考镜像。正式执行按：受众确认 → 第零步背景诊断 → 七步分析 → 输出 + 知识沉淀。

## 运行架构

```text
用户给文件/事件
 ↓
确认受众类型和核心问题
 ↓
第零步：知识需求诊断
 → 识别领域标签
 → 列出知识缺口假设
 → 定向查询 SQLite（domain_knowledge / historical_reference / current_state）
 → 获得"背景知识包"
 ↓
七步分析工作流
 第一步：异常值扫描
 第二步：让文件自己解释自己
 第三步：从外部资料中解释异常
 第四步：从排序读优先级
 第五步：用逻辑完整性做减法
 第六步：识别变与不变
 第七步：用外部知识校准
 ↓
输出解读报告（五段结构）
 +
输出知识沉淀包 → 写回 SQLite
```

## 关键规则（精简）

- 不做纯摘要，要给行动意义。
- 不能虚构数字；缺数据必须标记 `[知识缺口]` 或 `[待验证]`。
- 第三步是“异常级局部解释”，第七步是“全局校准”。
- 每次分析后都写回知识沉淀。
