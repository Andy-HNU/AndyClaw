# OpenClaw 信息检索手册 · 卷一：用户触发检索

> 本文档定义当你发现一个政策或时事线索时，如何告诉 openclaw 去找什么、怎么找、找到什么算够。
> 配套文档：《卷二：openclaw 自主巡检规则》

---

## 一、你只需要说一句话

openclaw 接受四种输入形式，不需要你整理好再说：

| 输入形式 | 示例 |
|---------|------|
| **关键词** | "发改委新出了个文" |
| **不完整标题** | "好像有个什么新能源政策" |
| **截图文字/转述** | "朋友说今天央行降息了" |
| **已知 URL** | 直接贴链接 |

收到后 openclaw 会自动执行"输入转化流程"，把碎片变成可分析的完整素材。

---

## 二、输入转化流程（openclaw 执行）

### Step 1：意图识别

判断这条输入属于哪种类型：

| 类型 | 判断依据 | 后续动作 |
|------|---------|---------|
| **明确政策文件** | 提到具体机构+文件性质 | 直接定点抓取 |
| **模糊政策线索** | 只有关键词或不完整信息 | 先搜索定位，再抓取 |
| **时事事件** | 涉及市场反应/突发事件/人事变动 | 多源交叉验证 |
| **已知 URL** | 用户直接提供链接 | agent-browser 直接抓取 |

---

### Step 2：检索执行

**政策文件检索路径**

```
用户说："发改委出了个文"
        ↓
Step 2a：搜索定位
  内置搜索 query：发改委 [当前月份年份] 新政策 OR 通知 OR 意见
        ↓
Step 2b：命中后用 agent-browser 抓取全文
  优先目标：
  - 国家发改委    https://www.ndrc.gov.cn
  - 国务院        https://www.gov.cn
  - 央行          https://www.pbc.gov.cn
  - 财政部        https://www.mof.gov.cn
  - 工信部        https://www.miit.gov.cn
  - 证监会        https://www.csrc.gov.cn
  - 国家统计局    https://www.stats.gov.cn
        ↓
Step 2c：确认是否找到目标文件
  找到 → 抓取全文，进入 Step 3
  未找到 → 告知用户，请求补充信息（时间/机构/关键词）
```

**时事事件检索路径**

```
用户说："好像伊朗有什么动作"
        ↓
Step 2a：内置搜索，获取事件轮廓
  query 构造规则：[地点/主体] + [动作关键词] + [近期/今天]
  示例：伊朗 军事 OR 制裁 OR 谈判 2026
        ↓
Step 2b：交叉验证（至少2个来源）
  中文优先源：财新、新浪财经、界面新闻
  英文补充源：Reuters、Bloomberg（仅在中文源信息不足时触发）
        ↓
Step 2c：判断是否达到"重大要闻"门槛（见第三节）
  达到 → 整理事件摘要，进入 Step 3
  未达到 → 告知用户："当前信息量不足以触发完整分析，已整理基本情况如下"
```

---

### Step 3：素材完整性检查

在交给分析框架之前，先检查素材是否够用：

**先做文件类型标注（强制）**
- A类：政策原文（部委正式通知/意见/规划等）
- B类：官方解读（新华社/人民日报/答记者问/吹风会解读）
- C类：二手报道

使用规则：
- A类可作为政策条款依据
- B类仅可作为方向信号，引用需标注“[解读稿引用，非原文条款]”
- C类不得作为分析与写库依据

```
政策文件检查清单：
  □ 原文全文已获取（非摘要）
  □ 发布机构和日期已确认
  □ 是否有配套解读（国务院/部委官方答记者问）

时事事件检查清单：
  □ 事件基本要素：时间、主体、动作、已知影响
  □ 是否有官方表态（政府/央行/国际机构）
  □ 市场反应数据（如有）：汇率/股指/大宗商品价格变动
```

**素材类型强制标注（进入分析前必须注明）**

| 类型 | 定义 | 引用权限 |
|------|------|---------|
| A类：政策原文 | 部委正式发布的通知/意见/规划/数据公报 | 可作为政策条款直接引用 |
| B类：官方解读 | 新华社通稿、答记者问、部委发布会实录 | 只能引用为"政策方向信号"，不能作为条款依据 |
| C类：二手报道 | 媒体报道、分析文章、转述内容 | 不得引用 |

每份素材必须在完整性检查结果中注明类型，例如：
- "《政府工作报告》全文：A类 ✅"
- "《开局之年，如何因地制宜发展新质生产力》：B类，政策方向参考"

**B类素材的特殊处理规则**：
- 分析中引用 B 类内容时，在对应判断后标注 [B类解读稿，非原文条款]
- 知识沉淀包的 source_quote 字段如来自 B 类，必须加注 [解读稿引用，非原文条款]
- 如果只有 B 类而无 A 类原文，在分析输出开头注明："本次分析基于官方解读稿，结论为方向性判断，待原文条款发布后需复核"

缺项处理：
- 政策文件缺原文 → 不进入分析，继续查找
- 时事事件缺官方表态 → 进入分析但在输出中标注 `[官方表态待确认]`
- 时事事件缺市场数据 → 进入分析，在知识缺口节标注

## 三、"重大要闻"判断标准

针对全球时事，不是所有事件都值得触发完整分析流程。openclaw 使用以下标准判断：

**必须触发分析（任意一条符合即可）**

| 标准 | 示例 |
|------|------|
| 影响全球大宗商品价格 ≥5% | 油价单日暴涨暴跌 |
| 涉及主要经济体央行政策变化 | 美联储意外降息/加息 |
| 地缘冲突升级影响全球供应链 | 霍尔木斯海峡封锁 |
| 中国直接相关的重大外部冲击 | 新一轮关税升级 |
| 全球系统性金融风险事件 | 主权债务危机、大型机构破产 |

**不触发分析，仅记录摘要**

| 情况 | 处理方式 |
|------|---------|
| 区域性冲突，未影响全球市场 | 写入 `current_state` 表备查 |
| 政治人事变动（非关键职位） | 忽略 |
| 单日市场波动（无明确原因） | 忽略 |
| 已分析过的事件有小幅进展 | 更新 `current_state` 表，不重新分析 |

## 四、政策检索的信源地图

### 4.1 一级信源（原文必须来自这里）

| 机构 | 网址 | 关注内容 |
|------|------|---------|
| 国务院 | https://www.gov.cn | 国务院常务会议、政策文件 |
| 国家发改委 | https://www.ndrc.gov.cn | 产业政策、投资政策、价格政策 |
| 中国人民银行 | https://www.pbc.gov.cn | 货币政策、利率、汇率 |
| 财政部 | https://www.mof.gov.cn | 财政政策、税收、债券 |
| 工业和信息化部 | https://www.miit.gov.cn | 制造业、新兴产业、数字经济 |
| 证监会 | https://www.csrc.gov.cn | 资本市场、上市规则 |
| 国家统计局 | https://www.stats.gov.cn | GDP、CPI、PMI 等数据 |
| 商务部 | https://www.mofcom.gov.cn | 外贸、外资、消费 |
| 科技部 | https://www.most.gov.cn | 科技创新、研发政策 |

### 4.2 二级信源（用于解读和背景补充，不作为原文依据）

| 来源 | 用途 |
|------|------|
| 新华社 | 官方通稿，政策定调的权威表述 |
| 人民日报 | 政策方向的官方背书 |
| 财新网 | 深度调查，政策执行层面的细节 |

| 中国证券报/上海证券报 | 资本市场政策信号 |

| 来源 | 语言 | 用途 |
|------|------|------|
| Reuters | 英文 | 地缘政治、大宗商品、国际机构动态 |
| Bloomberg | 英文 | 金融市场、央行动态 |
| 财新国际 | 中英 | 涉华国际新闻 |

## 五、常见输入场景示例

### 场景1：你看到一条朋友圈

> "听说央行今天降准了"

openclaw 执行：
```
意图识别：模糊政策线索（货币政策类）
搜索：中国人民银行 降准 [当前日期]
定点确认：https://www.pbc.gov.cn 新闻发布栏
找到公告 → 抓取全文 → 检查完整性 → 进入分析框架
未找到 → 回复："今日央行官网暂未发布降准公告，可能是市场传言，建议等待官方确认"
```

---

### 场景2：你贴了一个 URL

> "https://www.ndrc.gov.cn/xxgk/zcfb/tz/202603/xxxxx.html"

openclaw 执行：
```
意图识别：已知 URL
agent-browser 直接抓取全文
检查：发布机构、日期、文件类型
素材完整 → 进入分析框架
```

---

### 场景3：你说了一个模糊时事

> "好像中东又出事了"

openclaw 执行：
```
意图识别：模糊时事线索
内置搜索：中东 冲突 OR 局势 [本周]
交叉验证：财新 + Reuters
重大要闻判断：
  - 是否影响油价/供应链/中国外贸？
  判断为是 → 整理事件摘要，告知用户："发现[XXX事件]，已达到分析触发标准，是否进行完整分析？"
  判断为否 → 回复："当前中东动态为[简短描述]，暂未达到重大要闻触发标准，已记录备查"
```

## 六、openclaw 对用户的反馈规范

检索完成后，向用户汇报三件事，不多不少：

```
1. 找到了什么
   "已获取[机构][日期]发布的[文件名]全文"
   或"未找到明确文件，当前信息如下：[摘要]"

2. 素材是否完整
   "素材完整，可以进入分析"
   或"缺少[XXX]，已标注，分析结论可信度受限"

3. 是否需要进入完整分析
   "是否需要按[受众类型]视角进行完整解读？"
```

不要在检索阶段就输出分析结论。检索和分析是两个独立步骤。
