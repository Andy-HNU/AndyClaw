# OpenClaw 信息检索手册 · 卷二：自主巡检规则

> 本文档定义 openclaw 在无用户触发的情况下，如何主动扫描政策和时事信源、判断哪些内容值得推送、如何更新知识库。
> 配套文档：《卷一：用户触发检索》

---

## 一、巡检的目的

巡检不是为了"读完所有新闻"，而是为了两件事：

1. **发现值得分析的信号**：政策新出台、时事异常变化 → 推送给用户，等待触发分析
2. **更新知识库的当前状态**：即使不触发分析，也要让 `current_state` 表保持新鲜

两件事的标准不同，执行分开。

---

## 二、巡检节奏

| 巡检类型 | 频率 | 触发方式 |
|---------|------|---------|
| 中国政策快扫 | 每周一次（建议周一） | 定时任务或用户手动说"巡检" |
| 重大时事监测 | 每周一次（同上合并执行） | 同上 |
| 知识库状态更新 | 每次巡检结束后执行 | 随巡检自动触发 |
| 临时补扫 | 用户要求时 | 用户说"最近有没有什么新动向" |

> 所有巡检都在用户要求时才执行，不在后台静默运行。
> "每周一次"是建议频率，用户可以随时说"帮我巡检一下"手动触发。

## 三、政策巡检：信源与扫描规则

### 3.1 每周必扫信源（按优先级排序）

**第一层：国务院层面**

```
目标：https://www.gov.cn/zhengce/index.htm
扫描内容：
  - 国务院常务会议（议题列表）
  - 新发布的国务院文件（通知/意见/规划）
  - 国务院新闻办公室发布会
扫描深度：标题+首段，命中关键词后全文抓取
```

**第二层：核心经济部委**

```
按以下顺序逐一扫描首页新闻列表：
  1. 国家发改委  https://www.ndrc.gov.cn
  2. 中国人民银行 https://www.pbc.gov.cn/goutongjiaoliu/index.html
  3. 财政部      https://www.mof.gov.cn/zhengwuxinxi/
  4. 工信部      https://www.miit.gov.cn/jgsj/index.html
  5. 国家统计局  https://www.stats.gov.cn/sj/

扫描深度：标题扫描，命中触发词后进入全文
```

**第三层：重要数据发布**（按固定日历）

```
每月固定发布日期（openclaw 需记住）：
  - 每月15日前后：国家统计局 GDP/CPI/PPI/工业增加值
  - 每月末：财新PMI、官方PMI
  - 每季度末：央行货币政策执行报告
  - 每年3月：政府工作报告（全国两会期间）
  - 每年12月：中央经济工作会议
```

### 3.2 政策触发词清单

扫描标题时，遇到以下词汇即进入全文抓取：

| 触发词 | 信号类型 |
|-------|---------|
| 新出台/新发布/印发/颁布 | 新政策文件 |
| 调整/修订/修改 | 政策变化 |
| 首次/试点/探索 | 新方向信号 |
| 暂停/叫停/禁止 | 收紧信号 |
| 降准/降息/调整利率 | 货币政策变化 |
| 专项债/特别国债/赤字 | 财政政策变化 |
| 产业链/供应链/自主可控 | 产业政策信号 |
| 人工智能/新质生产力/新能源 | 重点赛道政策 |

遇到以下词汇**不触发**全文抓取（日常事务性内容）：

```
会见/访问/慰问/节日/表彰/竞赛/招聘/公示
```

## 四、时事巡检：信源与扫描规则

### 4.1 每周必扫信源

**中文源（每周必扫）**

```
1. 财新网首页    https://www.caixin.com
   重点：国际版块、经济版块
   扫描深度：标题+导语

2. 新浪财经要闻  https://finance.sina.com.cn/roll/index.d.html
   重点：国际财经、大宗商品
   扫描深度：标题扫描

3. 界面新闻      https://www.jiemian.com/lists/2.html
   重点：宏观经济、国际
```

**英文源（仅扫描标题，命中后中文摘要）**

```
1. Reuters 首页  https://www.reuters.com
   重点：Markets、World、Commodities 版块

2. IEA 新闻      https://www.iea.org/news
   重点：能源数据、储备动态
```

### 4.2 全球重大时事触发标准

**主动推送给用户的标准（任意一条符合）**：

| 触发条件 | 判断方法 |
|---------|---------|
| 油价/金价单周变动 ≥5% | 对比 current_state 表上周数据 |
| 美联储/欧央行政策会议结果 | 固定日历，每次必推 |
| 地缘冲突升级（涉及能源/航运通道） | 关键词：封锁/冲突升级/制裁升级/战争 |
| 美国对华政策重大变化 | 关键词：关税/制裁/出口管制/脱钩 |
| 全球金融系统性风险 | 关键词：破产/违约/系统性/蔓延 |
| 中国直接相关的重大外部事件 | 涉及中国主权/领土/供应链中断 |

**只更新知识库、不推送的情况**：

```
- 已知冲突/事件的常规进展（没有升级）
- 非关键国家的政治变动
- 单日市场波动（无明确触发原因）
- 经济数据小幅偏离预期（<1个百分点）
```

## 五、巡检执行流程

```
开始巡检
    ↓
【政策部分】
逐一访问第三节信源列表
  → 扫描标题
  → 命中触发词 → agent-browser 抓取全文
  → 未命中 → 跳过
    ↓
【时事部分】
逐一访问第四节信源列表
  → 扫描标题+导语
  → 对照重大要闻触发标准
  → 达标 → 内置搜索交叉验证（至少2个来源）
  → 未达标 → 判断是否需更新知识库
    ↓
【知识库更新】
将本周新出现的当前状态数据写入 current_state 表
（油价区间/汇率水平/最新政策定调/重要经济数据）
    ↓
【生成巡检摘要推送给用户】（见第六节）
```

### 工具使用规则

```
标题扫描阶段：内置搜索（快，低成本）
全文抓取阶段：agent-browser（精准获取官方全文）
  skill: https://clawhub.ai/MaTriXy/agent-browser-clawdbot
交叉验证阶段：内置搜索（多源对比）

原则：先用内置搜索判断是否值得抓，再用 agent-browser 抓全文
禁止：对每个信源都用 agent-browser 全量抓取（成本过高）
```

## 六、巡检摘要的输出格式

巡检结束后，向用户推送一份结构化摘要，不超过一屏：

```markdown
## 本周巡检摘要 [日期]

### 🔴 需要关注（建议触发分析）
- [政策/事件名称]：[一句话说明为什么值得分析]
  → 回复"分析[关键词]"可触发完整解读

### 🟡 已记录备查（暂不需要分析）
- [事件/数据名称]：[一句话摘要]

### 📊 知识库本周更新
- current_state 更新 X 条
  关键变动：[例：油价从XX升至XX / 央行维持利率不变]

### ⚪ 本周无异常的领域
- [领域列表]：无重大变化
```

**输出原则**：
- 🔴 项目最多3条，超过3条说明触发标准太宽，需要提高门槛
- 每条描述不超过2句话，细节留给完整分析
- 不在摘要中给出分析结论，只给"值不值得看"的判断

## 七、知识库更新规则

巡检中发现的当前状态数据，按以下标准决定是否写入：

| 数据类型 | 写入条件 | valid_until 设置 |
|---------|---------|----------------|
| 大宗商品价格区间 | 每周更新 | 下次巡检日期 |
| 央行政策利率 | 有变化时更新 | 下次政策会议日期 |
| 中国政策定调 | 重要文件出台时 | 下次重大会议日期 |
| 地缘冲突状态 | 有升级/降级时更新 | 2周后（需复查） |
| 重要经济数据 | 每次发布后更新 | 下次数据发布日期 |

**写入前必须确认**：
- 来源是否为一级信源或权威机构？
- 数据是否有明确的发布日期？
- 是否与现有记录冲突？（冲突时保留新值，在 context 字段注明旧值）

---

## 八、巡检质量自检

每次巡检结束前过一遍：

- [ ] 是否覆盖了所有第三节和第四节的必扫信源？
- [ ] agent-browser 是否只用于命中触发词后的全文抓取，没有滥用？
- [ ] 推送的🔴项目是否真正达到了触发标准，没有降低门槛？
- [ ] 知识库更新的每条记录是否有明确来源和 valid_until？
- [ ] 巡检摘要是否克制在一屏以内，没有变成新闻汇编？
